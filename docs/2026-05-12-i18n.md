项目基本完成后做 i18n，建议不要大改架构，也不要一开始引入复杂框架。对 Obsidian 插件来说，最稳方案是：

> **保留数据结构英文稳定，UI 文案做 i18n；用户内容不翻译；节点类型内部值不翻译；只翻译显示文本。**

你的项目可以这样做。

---

# 一、先明确哪些要翻译，哪些不要翻译

## 需要翻译

这些属于 UI 文案：

```txt
Text Node
Notebook Node
Open md
Create node
Delete node
Rename
Collapse
Expand
Search
Settings
Branch color
Follow Obsidian theme
```

还包括：

* 命令名称
* ribbon tooltip
* menu item
* setting tab 文案
* Notice 提示
* placeholder
* button
* context menu
* 空状态提示
* 错误提示
* 节点类型展示名

---

## 不要翻译

这些不要动：

```ts
type: "text"
type: "notebook"
renderMode: "plain"
renderMode: "inline-markdown"
branchColor
branchIndex
linkedFilePath
```

也就是说，`.naotu` 文件里仍然保存稳定英文 key：

```json
{
  "type": "notebook",
  "renderMode": "block-markdown"
}
```

显示时才翻译成：

```txt
Notebook Node / 笔记本节点
```

这样以后不会因为切换语言破坏文件兼容性。

# 二、推荐目录结构

建议加一个 `src/i18n`：

```txt
src/
  i18n/
    index.ts
    en.ts
    zh-CN.ts
    types.ts
```

不要一开始用很重的 i18next。你的插件是原生 DOM + TypeScript，自写一个轻量 `t()` 就够了。

# 四、类型约束

## `src/i18n/types.ts`

```ts
import { en } from "./en";

export type Locale = "auto" | "en" | "zh-CN";

export type TranslationSchema = typeof en;
```

这个做法的好处是：

> 英文文件是主 schema，其他语言必须跟它保持同样结构。

如果你想更严格，可以之后写一个脚本检查 zh-CN 是否缺 key。

# 五、i18n 核心函数

## `src/i18n/index.ts`

```ts
import { moment } from "obsidian";
import { en } from "./en";
import { zhCN } from "./zh-CN";
import type { Locale, TranslationSchema } from "./types";

const dictionaries: Record<Exclude<Locale, "auto">, TranslationSchema> = {
  en,
  "zh-CN": zhCN,
};

let currentLocale: Exclude<Locale, "auto"> = "en";

export function resolveLocale(locale: Locale): Exclude<Locale, "auto"> {
  if (locale !== "auto") {
    return locale;
  }

  const obsidianLocale = moment.locale();

  if (
    obsidianLocale === "zh-cn" ||
    obsidianLocale === "zh" ||
    obsidianLocale.startsWith("zh")
  ) {
    return "zh-CN";
  }

  return "en";
}

export function setLocale(locale: Locale) {
  currentLocale = resolveLocale(locale);
}

export function getLocale() {
  return currentLocale;
}

export function t(key: string, params?: Record<string, string | number>): string {
  const dict = dictionaries[currentLocale] ?? en;
  const fallback = en;

  let value = getNestedValue(dict, key) ?? getNestedValue(fallback, key) ?? key;

  if (params) {
    for (const [name, paramValue] of Object.entries(params)) {
      value = value.replaceAll(`{{${name}}}`, String(paramValue));
    }
  }

  return value;
}

function getNestedValue(obj: unknown, path: string): string | undefined {
  const parts = path.split(".");
  let current: any = obj;

  for (const part of parts) {
    if (!current || typeof current !== "object") {
      return undefined;
    }

    current = current[part];
  }

  return typeof current === "string" ? current : undefined;
}
```

# 六、插件设置里加入语言选项

你的设置可以这样：

```ts
export interface MindCanvasSettings {
  language: "auto" | "en" | "zh-CN";
  followObsidianTheme: boolean;
}
```

默认值：

```ts
export const DEFAULT_SETTINGS: MindCanvasSettings = {
  language: "auto",
  followObsidianTheme: true,
};
```

插件加载时：

```ts
import { setLocale } from "./i18n";

async onload() {
  this.settings = Object.assign({}, DEFAULT_SETTINGS, await this.loadData());

  setLocale(this.settings.language);

  this.addRibbonIcon("network", t("app.name"), () => {
    // open view
  });

  this.addCommand({
    id: "create-mindcanvas",
    name: t("action.create"),
    callback: () => {
      // ...
    },
  });
}
```

# 七、设置页使用 i18n

```ts
import { Setting } from "obsidian";
import { setLocale, t } from "./i18n";

new Setting(containerEl)
  .setName(t("settings.language"))
  .setDesc(t("settings.languageDesc"))
  .addDropdown((dropdown) => {
    dropdown
      .addOption("auto", t("settings.auto"))
      .addOption("en", t("settings.english"))
      .addOption("zh-CN", t("settings.chineseSimplified"))
      .setValue(this.plugin.settings.language)
      .onChange(async (value) => {
        this.plugin.settings.language = value as MindCanvasSettings["language"];
        await this.plugin.saveSettings();

        setLocale(this.plugin.settings.language);

        this.display();

        this.plugin.refreshOpenViews();
      });
  });
```

关键是：

```ts
this.display();
this.plugin.refreshOpenViews();
```

改语言后要重绘设置页和已经打开的脑图视图。

# 八、原生 DOM 中如何使用

之前硬编码：

```ts
button.setText("Open md");
button.setAttr("aria-label", "Open md");
```

改成：

```ts
button.setText(t("action.openMarkdown"));
button.setAttr("aria-label", t("action.openMarkdown"));
```

菜单：

```ts
menu.addItem((item) => {
  item
    .setTitle(t("menu.addChild"))
    .setIcon("plus")
    .onClick(() => this.addChildNode(node.id));
});
```

Notice：

```ts
new Notice(t("notice.nodeDeleted"));
```

placeholder：

```ts
inputEl.placeholder = t("placeholder.search");
```

节点类型显示名：

```ts
function getNodeTypeLabel(type: "text" | "notebook") {
  if (type === "text") return t("node.text");
  return t("node.notebook");
}
```

# 九、渲染层不要直接写中文/英文

你可以定一个规则：

> 除了测试数据和用户内容，源码里不允许出现直接 UI 文案。

坏例子：

```ts
nodeTypeEl.setText("笔记本节点");
```

好例子：

```ts
nodeTypeEl.setText(t("node.notebook"));
```

坏例子：

```ts
new Notice("File created.");
```

好例子：

```ts
new Notice(t("notice.fileCreated"));
```

# 十、动态参数

比如：

```txt
Deleted 3 nodes.
已删除 3 个节点。
```

语言文件：

```ts
notice: {
  nodesDeleted: "Deleted {{count}} nodes.",
}
```

中文：

```ts
notice: {
  nodesDeleted: "已删除 {{count}} 个节点。",
}
```

使用：

```ts
new Notice(t("notice.nodesDeleted", { count: 3 }));
```

---

# 十一、复数问题第一版可以先简单处理

英文严格来说要区分：

```txt
Deleted 1 node.
Deleted 3 nodes.
```

但第一版可以暂时写成：

```txt
Deleted {{count}} node(s).
```

或者避开复数：

```txt
Nodes deleted: {{count}}.
```

中文：

```txt
已删除节点：{{count}} 个。
```

如果以后用户多，再引入更复杂的 plural 机制。

# 十二、语言切换后要刷新哪些地方

改语言后至少刷新：

1. 设置页
2. 已打开的 MindCanvas view
3. 工具栏按钮 tooltip
4. context menu 下次打开即可
5. ribbon tooltip 通常需要重新注册才完全更新
6. command palette 里的命令名称可能需要重载插件才更新

这里要注意：

> Obsidian command palette 里的命令名一般是在 `addCommand()` 时注册的。语言切换后，不一定能即时更新命令列表。

建议设置页加一句说明：

```ts
settings: {
  restartRequiredForCommands: "Command names will update after reloading the plugin.",
}
```

中文：

```ts
restartRequiredForCommands: "命令面板中的名称会在重新加载插件后更新。",
```

# 十三、Manifest 和 README 怎么处理

`manifest.json` 里的插件名不要做动态 i18n。

建议保持：

```json
{
  "name": "MindCanvas"
}
```

中文名放 README：

```md
# MindCanvas 思维画布
```

README 可以这样：

```txt
README.md
README.zh-CN.md
```

插件市场主介绍用英文，中文用户进入仓库后能看到中文说明。

# 十四、i18n 改造顺序

项目快完成时，不要一次性全改。建议分 5 步。

## Phase 1：建立 i18n 基础

完成：

* `src/i18n/en.ts`
* `src/i18n/zh-CN.ts`
* `src/i18n/index.ts`
* `t()`
* `setLocale()`
* settings.language

---

## Phase 2：替换主界面文案

优先替换：

* toolbar
* context menu
* buttons
* node type labels
* empty state
* search placeholder

---

## Phase 3：替换设置页文案

包括：

* 设置项名称
* 设置项说明
* 下拉选项
* warning/help text

---

## Phase 4：替换 Notice 和错误提示

包括：

* 文件创建失败
* 文件打开失败
* 节点删除
* 不支持的文件类型
* Markdown 渲染错误

---

## Phase 5：测试语言切换

测试：

* Obsidian 英文界面
* Obsidian 中文界面
* 插件设置 Auto
* 手动选择 English
* 手动选择 简体中文
* 暗色/亮色下中文字号是否拥挤
* Notebook Node 里的 Markdown 用户内容不被翻译

# 十五、CSS 也要考虑中文

中英文长度不同，中文更紧凑，英文按钮更长。

建议不要写死按钮宽度：

```css
.mindcanvas-button {
  min-width: 0;
  white-space: nowrap;
}
```

节点类型 chip：

```css
.mindcanvas-node-chip {
  max-width: 100%;
  overflow: hidden;
  text-overflow: ellipsis;
  white-space: nowrap;
}
```

中文环境下可以稍微调 line-height：

```css
.mindcanvas-root {
  line-height: 1.45;
}
```

# 十六、最终给开发模型的执行提示词

```md
为 Obsidian 插件 MindCanvas 添加轻量 i18n 支持。

要求：
1. 插件使用原生 DOM + TypeScript，不使用 React。
2. 不引入 i18next，先实现轻量 t() 函数。
3. 新增 src/i18n/en.ts、src/i18n/zh-CN.ts、src/i18n/index.ts、src/i18n/types.ts。
4. 英文 en.ts 作为主 schema。
5. 支持语言：auto、en、zh-CN。
6. auto 模式下通过 Obsidian 的 moment.locale() 判断当前语言，中文环境使用 zh-CN，否则使用 en。
7. 所有 UI 文案使用 t("xxx.xxx") 获取。
8. 支持简单参数插值，例如 t("notice.nodesDeleted", { count: 3 })。
9. 用户内容、Markdown 内容、.naotu 文件内容不翻译。
10. 节点内部数据仍使用稳定英文值，例如 type: "text"、type: "notebook"。

15. 替换 toolbar、context menu、settings、notice、placeholder、button、node type label 中的硬编码文案。
```

我建议你最终的策略是：

> **内部数据全英文稳定，界面文案全部走 `t()`，用户内容完全不碰。**
