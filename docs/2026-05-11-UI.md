
**从中心节点出发的一级分支拥有自己的分支色；该分支下的连线、节点边框、折叠符号、选中路径等都继承这个颜色。**



# 定义颜色系统


# 二、新的颜色优先级

我建议改成这套规则：

| 颜色类型         | 用途                          | 是否核心        |
| ------------ | --------------------------- | ----------- |
| 分支色          | 表示从中心节点出来的主分支归属             | 核心          |
| 交互蓝          | 表示当前 selected / editing     | 核心，但只在交互时出现 |
| 状态色          | warning / error / success 等 | 非核心，先不做或只预留 |
| Obsidian 主题色 | 背景、文字、面板                    | 环境层         |

也就是：

> **分支色负责“我属于哪一支”；蓝色负责“我现在被操作”；Obsidian 负责“我处在哪个主题环境”。**

---

# 三、分支色应该影响哪些元素

对于从中心节点出来的每个一级分支，设定一个 `branchColor`。

这个颜色应该影响：

## Text Node

* 连线颜色
* 节点边框色
* 折叠 `+ / -` 颜色
* hover 时的浅色背景
* selected 时可以和蓝色共存，但不要冲突

## Notebook Node

* 左侧色条
* 顶部细线
* 卡片边框色
* 连接线颜色
* 卡片标题旁的小圆点
* resize 角标颜色，可以可选

## 连线

* 主干线使用分支色
* 深层子线继承同一个分支色
* 子线透明度可逐级降低
* hover / active path 时加粗或提高透明度

---

# 四、分支色和选中蓝怎么共存

这是关键。

如果分支边框已经有颜色，选中时再用蓝色，会出现两个问题：

1. 用户不知道蓝色和分支色谁更重要
2. 选中节点会丢失原来的分支归属感

所以我建议：

## 默认态

节点边框 / 连线使用分支色。

## hover 态

提高当前分支色透明度或线宽。

## selected 态

不要把边框直接改成蓝色。
更好是：

* **节点原边框仍保留分支色**
* 外面增加一圈蓝色 focus ring
* 或者使用蓝色阴影环

这样用户同时知道：

> 这个节点属于橙色分支，并且当前被选中。

示例：

```css
.mindmap-node.is-selected {
  box-shadow: 0 0 0 3px var(--mindmap-active-soft);
}
```

而节点自己的边框仍然是：

```css
border-color: var(--branch-color);
```

这比直接把边框换成蓝色更好。

# 六、分支色分配规则

从中心节点出发的一级子节点按顺序分配颜色：

```ts
const BRANCH_COLORS = [
  "var(--mindmap-branch-1)",
  "var(--mindmap-branch-2)",
  "var(--mindmap-branch-3)",
  "var(--mindmap-branch-4)",
  "var(--mindmap-branch-5)",
  "var(--mindmap-branch-6)",
  "var(--mindmap-branch-7)",
  "var(--mindmap-branch-8)",
];

function getBranchColor(index: number) {
  return BRANCH_COLORS[index % BRANCH_COLORS.length];
}
```

如果一级分支超过 8 个，就循环使用。
不过建议 UI 里尽量不要让用户在一个中心主题下展开太多一级分支，否则脑图本身也会变乱。

# 七、节点如何继承分支色

每个节点需要知道自己属于哪个一级分支。

数据结构可以加一个运行时字段：

```ts
type MindmapNode = {
  id: string;
  type: "text" | "notebook";
  parentId?: string;
  children?: string[];

  branchIndex?: number;
  branchColor?: string;
};
```

但更建议 `branchIndex / branchColor` 不一定持久化，而是在 layout/render 阶段从树结构推导：

```ts
function assignBranchMeta(root: MindmapNode, nodesById: Map<string, MindmapNode>) {
  root.children?.forEach((childId, index) => {
    const color = getBranchColor(index);
    assignBranchToSubtree(childId, index, color, nodesById);
  });
}

function assignBranchToSubtree(
  nodeId: string,
  branchIndex: number,
  branchColor: string,
  nodesById: Map<string, MindmapNode>
) {
  const node = nodesById.get(nodeId);
  if (!node) return;

  node.branchIndex = branchIndex;
  node.branchColor = branchColor;

  node.children?.forEach((childId) => {
    assignBranchToSubtree(childId, branchIndex, branchColor, nodesById);
  });
}
```

这样有一个好处：

> 当用户调整一级分支顺序时，颜色可以自然重新分配；或者你也可以以后支持“锁定分支色”。

# 八、CSS 使用方式

渲染每个节点时，把分支色写到 CSS variable：

```tsx
<div
  className={`mindmap-node ${node.type === "text" ? "text-node" : "notebook-node"}`}
  style={{ "--branch-color": node.branchColor } as React.CSSProperties}

```

连线也一样：

```tsx
<path
  className="mindmap-link"
  style={{ "--branch-color": link.branchColor } as React.CSSProperties}
  d={path}
/>
```

# 十二、折叠 `+ / -` 也用分支色

保持现在的视觉形式即可，但颜色继承分支：

```css
.mindmap-collapse-control {
  color: var(--branch-color);
  opacity: 0.85;
  font-weight: 700;
}

.mindmap-collapse-control:hover {
  opacity: 1;
}
```

这样 `+ / -` 就不再是孤立符号，而是这条分支的一部分。

# 十三、中心节点怎么处理

中心节点不属于任何一级分支，所以它不应该使用分支色。

中心节点可以使用中性强调：

```css
.mindmap-node.root-node {
  background: var(--background-secondary);
  border: 1.5px solid var(--background-modifier-border);
  box-shadow: 0 8px 24px rgba(0, 0, 0, 0.12);
  font-weight: 700;
}
```

如果中心节点被选中，再出现蓝色 focus ring：

```css
.mindmap-node.root-node.is-selected {
  box-shadow:
    0 0 0 3px var(--mindmap-active-soft),
    0 8px 24px rgba(0, 0, 0, 0.12);
}
```

# 十四、最终建议改成这句话

你现在的设计原则可以更新为：

> **画布背景跟随 Obsidian；节点类型分为 Text Node 和 Notebook Node；从中心节点出发的每个一级分支拥有独立分支色；该分支下的连线、节点边框、折叠符号继承分支色；蓝色仅用于选中、聚焦和编辑。**

这比之前的状态色系统更贴合脑图。

---

# 十五、给开发模型的更新提示词

```md
更新脑图插件 UI 色彩系统。

取消以 warning/error/success 为核心的状态色系统，改为以 branch color 为核心。

设计规则：
1. 画布背景跟随 Obsidian，使用 var(--background-primary)。
2. 节点类型正式分为 Text Node 和 Notebook Node，节点类型由用户选择。
3. 从中心节点出发的每一个一级分支分配一个 branch color。
4. 一级分支下面的所有后代节点继承同一个 branch color。
5. 如果一级分支数量超过调色板数量，则循环使用颜色。
6. branch color 用于该分支的连线、节点边框、折叠 + / -、Notebook Node 左侧色条或顶部强调线。
7. 不要让 Notebook Node 整张卡片大面积染色，只用边框、色条、角标等小面积区域表达分支归属。
8. Text Node 默认保持轻量，内部节点可有轻卡片；边框和 hover 使用 branch color。
9. 蓝色只用于 selected / active / editing，不表示 root、Notebook Node 或图片节点。
10. selected 状态不要覆盖 branch color，保留 branch color 边框，在外层增加蓝色 focus ring。
11. Root Node 不属于任何 branch，使用中性边框；选中时才出现蓝色 focus ring。
12. 暗色主题下保持同一套 branch color 语义，只调整颜色明度和对比度。
```

一句话总结就是：

> **状态色可以先弱化，分支色才是脑图的主色彩系统。**
